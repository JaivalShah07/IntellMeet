**IntellMeet** is an enterprise-grade meeting platform that combines video collaboration, real-time chat, AI meeting intelligence, task management, and analytics — built for internship/portfolio submission and PPO evaluation (Zidio / LogicVeda).

> Turn meetings into actionable outcomes: AI summaries, extracted tasks, and team dashboards — reducing follow-up work by 40–60%.

## Quick Start

### Prerequisites

- Node.js 20+
- MongoDB instance (Atlas cluster or local install)

### 1. Install Dependencies

Install all dependencies for both frontend and backend using the root helper script:

```bash
npm run install:all
```

### 2. Configure Environment

Copy env templates and set your keys:

```bash
# In root directory:
cp backend/.env.example backend/.env
cp frontend/.env.example frontend/.env
```

Ensure `backend/.env` is configured with your MongoDB Atlas or local URI and your Google Gemini API key:
```env
MONGODB_URI=your_mongodb_connection_string
GEMINI_API_KEY=your_gemini_api_key
```

### 3. Seed Demo Data

Populate the database with test users and meetings:

```bash
npm run seed
```

### 4. Run Full Stack

Start both development servers concurrently:

```bash
npm run dev
```

Open http://localhost:5173 to test.

## Troubleshooting

- **Invalid Credentials / Connection Error:** Ensure your `MONGODB_URI` in `backend/.env` is correct and your database is accessible.
- **AI Analysis Fails:** Verify that a valid `GEMINI_API_KEY` is present. If missing, the backend logs a warning and falls back to a clean mock response.
- **Port Conflict:** Make sure port `5000` (API) and `5173` (Vite) are not occupied.

## Security Best Practices

- Secure password storage using `bcrypt` (12 rounds).
- HTTP-only authentication tokens / JWT payload authorization.
- Request rate-limiting implemented on authentication endpoints to prevent brute force attacks.
- Robust header safety through `Helmet` middleware.
- Restricted Cross-Origin Resource Sharing (CORS).
